## 🔧 Configuration

### Setting Up Groq API

1. **Get API Key**
   - Visit https://console.groq.com
   - Create free account (no credit card required)
   - Navigate to "API Keys" section
   - Create and copy your API key (starts with `gsk_`)

2. **Configure** (`backend/.env`)
   ```bash
   GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   GROQ_MODEL=llama-3.1-8b-instant
   LLM_PROVIDER=groq
   ```

3. **Restart Services**
   ```bash
   # Terminal 1
   cd backend && python -m uvicorn app.main:app --reload --port 8000
   
   # Terminal 2
   cd backend && celery -A app.workers.celery_app worker --loglevel=info
   ```

## 🎯 Test Generation Examples

### Input Function
```python
def divide(a, b):
    """Divide a by b"""
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b
```

### Generated Tests (via Groq LLM)
```python
import pytest
from module import divide

def test_divide_normal_case():
    """Test divide with positive numbers"""
    assert divide(10, 2) == 5.0

def test_divide_by_zero():
    """Test divide raises error for zero divisor"""
    with pytest.raises(ValueError, match="Cannot divide"):
        divide(10, 0)

def test_divide_negative_numbers():
    """Test divide with negative numbers"""
    assert divide(-10, 2) == -5.0

def test_divide_floats():
    """Test divide with float inputs"""
    assert divide(10.5, 2.0) == 5.25

def test_divide_zero_dividend():
    """Test divide zero by non-zero"""
    assert divide(0, 5) == 0.0
```

## 🎓 How It Works (Technical Details)

### Prompt Engineering

The system creates context-aware prompts:

```python
prompt = f"""
You are an expert Python test engineer. Generate comprehensive unit tests for:

Function Code:
```python
{function_code}
```

Edge Cases Identified:
- Zero division
- Negative numbers
- Float precision
- Boundary values

Requirements:
1. Use pytest
2. Include tests for all edge cases
3. Use descriptive names
4. Include docstrings

Generate ONLY test code, no explanations.
"""
```

### Response Processing

1. Groq returns generated test code as string
2. System stores with metadata:
   - `generated_by`: "groq" (or "demo")
   - `test_type`: "comprehensive" or "edge_cases"
   - `target_function`: function name
   - `content`: actual test code
3. Results stored in database with job
